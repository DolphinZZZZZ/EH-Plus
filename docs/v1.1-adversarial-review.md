## 一、已实现（与规划一致，本轮抽查证据通过）

以下能力经逐项核对与规划一致，仅列要点：

- **双向交错预加载**：真实 `#next`/`#prev` 链接、下/上/下二/上二交错、`preloadAhead` 总槽位、缺 prev 补读当前页、不用 token 拼页号。
- **每次 fetch 前页面忙闲门控**：`EHPE_PAGE_NETWORK_ACTIVITY_QUERY` + 250ms 轮询；空闲先补存再放行；`preloadEnabled=false` 只跳过新预加载。
- **多页面调度与队列去重**：全局并发槽共享、聚焦标签优先（tabs.onActivated/windows.onFocusChanged）、观测对账 ≥1s 节流、已加载页只续链不抓图、加载中降级队尾、会话过期清队、对账失败回退默认交错顺序、槽位等待 120s 强制放行。
- **nl 换源体系**：guard 脚本注入顺序在 cache-first 之前、地址栏 nl 清理、`nl()` / `onerror` / `onclick` 接管、替换式重试（≤3 次、去重 URL）、`nl-retry-bypass` 全链路（含 SW 排除带 nl 的 URL、release 5×1200ms 重试）。
- **cache-first 超时预算全部对数**：3500/2400/1500/600/900/5000/20000/30000/32000ms 与规划一字不差；ensure/release 单调序号；webNavigation 只释放不抢装；同文档翻页 pageKey 生命周期与 `cached-url-kept`。
- **内置自动翻页 /s/**：Super-preloader DOM 形态与 6 箭头资源、白底分隔条、克隆片段清理 nl 内联入口、启动先让位 cache-first 并等 DNR 释放、Aplus 预构建 + 游离失败图补发 error、分隔条箭头灰亮重算；isolated/MAIN 双 world 判定统一（`hasBuiltInAutoPagerDom`）。
- **浮窗**：解耦启动、紧凑收起、双击 1s 冷却、命中缓存状态行优先、实时图片加载状态行（2 条、8s 隐藏）、并发/排队显示勾选、细节行按当前画廊 gid 口径。
- **账号/签到**：固定 e-hentai.org 数据源、kGP×1000、限额三色、0 GP 置灰、真实消耗差值展示规则、0→1 页立即刷新 + 600s 周期 + activeTabs ≤80 去 query/hash；Dawn 两开关默认关、UTC 03:00 后、后台次数只计真 Dawn 成功、`7/4已签到` 格式。
- **浏览历史**：`recordKind:'history'` 轻量记录、EhViewer 风格独立页、双入口走 `EHPE_OPEN_HISTORY`、封面回退（gid:1 内部查询、并发 4、会话缓存）、galleryMeta 上报不加次数。
- **日志**：30 天/100MB 真实默认、`ehpe_live_logs` 分 key、4MB 安全上限 + 2000 条保险、请求诊断字段、Debug 全文 24h。
- **迁移**：逐条回读校验、等价跳过（断点续迁）、目录间先水合再写、后台执行 + 进度轮询 + 取消（条目边界、不删旧缓存）。
- **打包与版本**：manifest/package/popup/后台/CHANGELOG 全部 1.0.0；权限最小化核查通过（无 notifications/unlimitedStorage）；构建瘦身生效（包 49 文件、zip ≈0.2MB）；dist 与 Downloads 哈希一致；`node --test` 102 项全过；废弃 userscript 协议字符串全局清零。

## 二、v1.1 预计优化项目

### P1（高优先级：行为与规划相悖或数据风险）——已全部于 1.0.0 修复

修复落点：1 → `stripImageFromRecord`/目录 `stripImages`（按天清图只剥图片体，保留索引与统计）；2 → `planImageCacheLimitCleanup` 接受 `protection` 并跳过受保护记录；3 → content 侧无 pageKey 候选按 `url:` 键上报、SW 写 TEMPORARY resource-only 记录；4 → 上限按总占用换算（`分配上限 − 日志 − 其他` 作为图片字节预算），"数量上限清理"确认为规划预留规则（规划 §9 已加注，1.0.0 无该设置故不实现）。

| # | 项目 | 现状（证据） | 目标 |
| --- | --- | --- | --- |
| 1 | **按天清理图片改为只删图片体、保留元数据** | 规划 §8 要求按天清图只删 `imageBlob`/目录图片文件，保留页面索引与统计；实际 `planRuntimeCleanup` 将整条页记录放入删除列表并 `deleteMany` 整键删除（shared/cleanup.js L363-368、service-worker.js L4638） | 清理路径改为剥离图片体、保留 metadata stub；目录模式只删 `images/` 文件并更新索引 |
| 2 | **大小上限清理豁免受保护图片** | 规划 §9 要求上限淘汰不删受保护；`planImageCacheLimitCleanup`（shared/cleanup.js L124-149）排序淘汰未调用 `isProtectedImage` | 淘汰序过滤受保护记录；受保护占用超限时按既有停写规则处理 |
| 3 | **打通无 gid:pageNo 的临时补存路径** | 规划 §953 要求只有 H@H URL 的图片写临时 resource-only 缓存；实际 content 侧 `collectExternalImageCacheFillItems` 直接跳过无 pageKey 项（content-script.js L1395），SW 侧 `cacheFillExternalImage` 硬拒绝（service-worker.js L2352），临时落库路径存在但被拒 | 上报无 pageKey 候选 + 允许 TEMPORARY 落库，遵守"不入永久索引、全关清理、超限先清临时" |
