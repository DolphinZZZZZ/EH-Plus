# EH＋

E-Hentai / ExHentai 阅读增强扩展（Chrome Manifest V3）。

EH＋ 在不改变站点原有使用方式的前提下，为 `/g/` 画廊页与 `/s/` 阅读页提供图片预加载、本地缓存、自动翻页、账号工具与数据统计，让翻页更快、加载更稳、重复请求更少。

> 本项目免费开源，GitHub 是唯一官方发布来源；任何其他渠道的付费出售均为第三方倒卖行为，与本项目无关。

## 主要功能

### 阅读加速
- **图片预加载**：阅读页按“下一页、上一页、下下页……”双向交错顺序自动预取后续页面，页数与全局并发数可调；多开页面共享并发额度，聚焦标签页优先，已在其他页面加载过的图片自动去重不重复下载。
- **缓存优先显示（cache-first）**：开启后，进入阅读页时优先使用本地缓存展示图片，未命中时快速恢复原图请求，不阻塞正常浏览。
- **换源重试**：接管 EH/EX 官方 `nl()` 换源机制，图片加载失败时后台自动换源重试，地址栏不再残留 `?nl=` 参数、页面不再整页刷新。
- **内置自动翻页**：`/s/` 阅读页与 `/g/` 画廊页均支持滚动到底自动拼接下一页，拼接图片优先走本地缓存。

### 本地缓存
- **两种存储模式**：默认浏览器 IndexedDB；也可授权本地目录，把图片按 `images/<gid>/` 结构存成真实文件。
- **存储迁移**：IndexedDB 与授权目录之间迁移，逐条校验、支持断点续迁、可取消、实时显示进度。
- **缓存治理**：按天数清理（可保护高频访问的图片与画廊）、重复图片自动合并去重、临时缓存随全部 EH/EX 页面关闭自动回收。
- **容量与统计**：存储占用、命中率、访问计数一目了然，日志保留天数与大小上限可配置。

### 页面浮窗
- 可拖拽、可折叠的悬浮面板：状态点与预加载矩阵、实时图片加载状态（命中缓存 / 换源重试 / 失败可见）、当前并发与排队数、迷你统计。
- 全部设置就地生效：预加载、缓存、自动翻页、清理、迁移、日志、颜色自定义、中英双语切换。

### 账号与签到
- **账号状态**：图片限额、Credits、GP、Hath 等状态刷新，打开 EH/EX 页面期间后台定时保持更新。
- **限额重置**：二次确认后提交官方 Reset Quota 请求，并显示实际消耗。
- **Dawn 每日签到**：页面打开时自动检查，或无需开页的后台定时签到，奖励明细可见。

### 浏览历史
- 独立浏览历史页：记录 `/g/` 与 `/s/` 访问，支持来源筛选、按最近访问 / 访问次数排序、单条删除与清空，上限可自定义，且与图片缓存互不影响。

### 开放接口
- 提供合作式缓存查询消息（`EHPLUS_COOPERATIVE_CACHE_QUERY`）：其他扩展或页面脚本可查询 EH＋ 的本地缓存并直接复用已缓存图片，避免重复请求。

## 安装

1. 从 [Releases](../../releases) 下载 `EH＋-extension-v<版本>.zip` 并解压。
2. 打开 `chrome://extensions/`，开启右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”，选择解压后的文件夹。
4. 打开或刷新任意 E-Hentai / ExHentai 画廊页或阅读页，左上角出现 EH＋ 浮窗即安装成功。

更新方式：从 Releases 下载新版本覆盖加载。扩展内“关于”分组提供更新检查。

## 从源码构建

需要 Node.js ≥ 20。

```sh
npm test                 # 运行全部自动化测试
npm run build:extension  # 测试 + 打包，产物输出到 dist/ 并复制到 Downloads
```

## 权限说明

| 权限 | 用途 |
| --- | --- |
| `storage` / `alarms` | 保存设置与运行状态；定时签到、账号刷新与缓存维护 |
| `tabs` / `webNavigation` | 判断 EH/EX 页面开关状态、聚焦标签优先调度 |
| `declarativeNetRequest` | 仅在开启缓存优先时，对当前阅读页图片请求做本地缓存抢占 |
| `downloads` | 下载 GitHub Release 更新包 |
| host 权限 | 仅限 `e-hentai.org`、`exhentai.org` 与 `*.hath.network` 图片服务器 |

所有数据（缓存、设置、日志、统计）均保存在本地，无任何遥测或第三方上报。网络请求仅发往 EH/EX 官方页面与其图片服务器；账号相关操作全部使用浏览器已登录的会话。
